Incident Response Briefing

ATTN SECURITY ARCHITECTS: You have been hired to fix the broken permissions at Apex Solutions. Apply the Principle of Least Privilege (PoLP). Give users exactly what they need to do their jobs, and absolutely nothing more.

The Golden Rule of Windows NTFS: Explicit "Deny" permissions will ALWAYS override "Allow" permissions.

The Employees:

  • Alice: The CEO. She needs to read financials, but shouldn't be modifying IT systems or HR records.
  • Bob: HR Director. Handles payroll and sensitive employee data.
  • Charlie: Lead Developer. Writes the code for the company's main software product.
  • Dave: IT Administrator. Keeps the servers running and manages backups.
  • Eve: Marketing Intern. Creates social media posts. Should have very restricted access.

The Folders:

  • C:\Shares\Public - For company-wide memos and casual sharing.
  • C:\Shares\HR_Records - Highly sensitive salaries and write-ups.
  • C:\Shares\SourceCode - The company's proprietary software code.

Assign Users to Security Groups

Map each employee to their correct security group. (In a real Windows environment, applying permissions to Groups is much more efficient than applying them to individual Users).

Configure Access Control Lists (ACL)

Set the NTFS permissions for each group on the network shares.
